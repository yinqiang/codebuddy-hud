## Phase 4 实战集成完成

### 性能优化
- **Git 并行执行**: 所有 git 命令改用 Promise.all 并行，不再串行
- **Fast dirty check**: essential/minimal 预设用 `git diff --quiet HEAD` 替代 `git status --porcelain`
  - 大仓库（UE5）从 ~1128ms 降到 ~60ms
  - `git diff --quiet` 只检查是否有改动，不列举文件，快得多
- **配置感知 git**: 传 GitOptions 跳过不需要的 ahead/behind/file stats 命令
- **增量 transcript 解析**: `parseTranscriptIncremental` + 字节级 tail read
  - >256KB 文件只读尾部 200 行
  - 26MB transcript: 1470ms → 3ms
- **Git 缓存 TTL**: 3s（statusline 300ms 刷新周期，~10 次命中）
- **性能分析**: `CODEBUDDY_HUD_PROFILE=1` 环境变量，输出各阶段耗时到 stderr
